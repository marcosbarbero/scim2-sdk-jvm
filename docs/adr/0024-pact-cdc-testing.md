# ADR-0024: Pact.io for Consumer-Driven Contract Testing

## Status
Accepted

## Context
The SDK provides both a SCIM client and server. We need to verify that:
1. The client's expectations match what any RFC-compliant SCIM server provides
2. Our server implementation satisfies what SCIM clients expect

Hand-rolled contract tests (ScimApiContractTest) validate RFC compliance but don't provide Consumer-Driven Contract (CDC) benefits: consumer/provider verification, contract sharing, backwards compatibility checks.

## Decision
Use [Pact.io](https://pact.io) for CDC testing:
- **Consumer tests** (scim2-sdk-client): Define what the ScimClient expects from a SCIM server
- **Provider tests** (scim2-sdk-test): Verify our server satisfies those expectations
- Pact files generated by consumer tests, verified by provider tests
- Keep existing ScimApiContractTest for RFC-level compliance (not CDC)

Chose Pact over Spring Cloud Contract because:
- Pact is language/framework agnostic (matches our framework-agnostic core)
- Pact has broader adoption for API contract testing
- Spring Cloud Contract would only work for the Spring module

## Consequences
- Client and server contracts are machine-verified
- Breaking changes detected before release
- Pact files serve as living API documentation
- CI can verify contracts on every PR
